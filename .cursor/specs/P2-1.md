# P2-1 — Frontend: React Query + Dashboard Refactor

## Hedef
Dashboard.tsx'teki tek büyük `useEffect` + çok sayıda `useState` kombinasyonunu
`@tanstack/react-query` v5 hook'larıyla değiştir. Kazanımlar:

- Otomatik background refetch (window focus, stale)
- Bireysel loading/error durumları
- Retry mantığı built-in
- Gelecekte diğer sayfaların hook'larını da buradan türetme imkânı

## Kapsam (P2-1 sınırları)

Sadece **Dashboard.tsx** refactor edilir. Diğer sayfalar P2 ilerleyen görevlere bırakılır.

## Yapılacaklar

1. `@tanstack/react-query@^5` yükle (React 19 uyumlu).
2. `frontend/src/lib/queryClient.ts` → `QueryClient` tanımı (staleTime 30 s, gcTime 5 dk, retry 1).
3. `frontend/src/main.tsx` → `<QueryClientProvider>` ekle.
4. `frontend/src/hooks/useDashboardQueries.ts` → Dashboard'a özel hook'lar:
   - `useDailySales(startDate, endDate, enabled)`
   - `useSalesRecords(startDate, endDate, enabled)`
   - `useForecastResults(enabled)`
   - `useProducts(enabled)`
   - `useFinanceSummary(startDate, endDate, enabled)`
   - `useActiveModules(enabled)`
   - `useUsers(enabled)`
5. `Dashboard.tsx` → `useEffect`/`useState` bloğunu hook çağrılarıyla değiştir;
   `loading = birden fazla sorgu yükleniyor`, `error = ilk hata mesajı`.

## Kabul kriteri

- `npm run build` hatasız tamamlanır.
- `npm test` mevcut testler geçer.
- Dashboard'da veriler aynı şekilde görünür (görsel regresyon yok).
- Yeni isim: her sorgu için ayrı `isLoading` — `loading` tek boolean yerine.

## Test

Frontend test altyapısı (vitest) ile smoke test: `QueryClientProvider` olmadan
render edilince hata vermemeli (queryClient mock ile).
